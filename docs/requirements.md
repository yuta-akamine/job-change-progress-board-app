# 就活ステータス管理ボードアプリ 要件定義（MVP）

## 1. 目的（Why）
本アプリは、就職・転職活動における応募企業・選考状況を
「一目で把握できる状態」にすることを目的とする。

本プロジェクトは以下を重視する：
- 実際の課題を解決する実用性
- フルスタック（Laravel + React + Inertia）の統合力
- AI駆動開発における設計主導・制御能力の実証

---

## 2. スコープ（What）

### 2.1 対象ユーザー
- 本アプリの利用者は **ログインした本人のみ**
- マルチユーザー・チーム利用は想定しない

### 2.2 提供する機能（MVP）

#### 認証
- ユーザー登録
- ログイン / ログアウト
- 認証は Laravel Breeze を利用する

#### 応募情報管理
ユーザーは以下の情報を管理できる：

- 企業名（必須）
- 職種（任意）
- 選考ステータス（必須）
- 面接日（任意）
- メモ（任意）

CRUD 操作：
- 作成
- 一覧表示
- 更新
- 削除

#### 選考ステータス
選考ステータスは以下に固定する：


- カジュアル面談
- 応募予定
- 書類選考
- 筆記試験
- 面接（一次〜最終）
- 内定
- 辞退/見送り

#### カンバン表示
- 応募情報はステータスごとにカンバン形式で表示する
- 各応募はカードとして表示する

#### ドラッグ＆ドロップ
- カードをドラッグして列を移動できる
- 移動時にステータスが即座に更新され、DBに保存される

---

## 3. 非機能要件（Constraints）

### 3.1 技術スタック（固定）
以下の技術スタックは **変更不可** とする。

- Backend: Laravel 12 (PHP)
- Frontend: React
- SPA連携: Inertia.js
- Styling: Tailwind CSS
- DB: MySQL
- 開発環境: Laravel Sail (Docker)

### 3.2 開発条件
- AI（Cursor）を利用したAI駆動開発を行う
- ただし、設計・要件の最終決定権は私(人間)が持つ
- 8時間以内で「実装・動作確認・デプロイ」まで完了させる

### 3.3 セキュリティ
- 認証必須（未ログインでは操作不可）
- 応募情報は **自分のデータのみ操作可能**
- CSRF 対策は Laravel 標準機構を利用する

---

## 4. 明確に「やらないこと」（重要）

以下は **今回のMVPで以下は実装しない**。

- 複数ユーザー間でのデータ共有
- チーム・企業単位での管理機能
- メール通知・リマインダー
- ファイルアップロード（履歴書等）
- 高度な検索・フィルタリング
- 自動テストの網羅的実装
- 権限ロール（管理者・一般ユーザー等）

---

## 5. 完成の定義（Definition of Done）

以下をすべて満たした状態を「完成」とする。

- ユーザー登録 → ログイン、ログアウトができる
- 応募情報を1件以上登録できる
- カンバン形式で一覧表示される
- ドラッグ＆ドロップでステータスを変更できる
- 変更がDBに正しく保存される
- スマホ画面で最低限崩れずに表示される
- 本番環境にデプロイされ、URLでアクセス可能
